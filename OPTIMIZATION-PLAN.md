# DUSKFALL — План глобального дебаггинга и оптимизации

## Принципы (КРИТИЧНО)
1. **НЕ ТЕРЯТЬ КАЧЕСТВО** — все функции должны работать как раньше
2. **НЕ УДАЛЯТЬ нужные компоненты** — перед удалением проверить использование
3. **Коммитить после каждого этапа** — чтобы не потерять работу
4. **Сохранить все 50+ механик** — conditions, AoE, flanking, spell slots, equipment, crafting, bestiary, spellbook, items, i18n, audio, TTS, auth, saves, dungeons, etc.

## Этап 1: Полный аудит (read-only)
- Список всех файлов и их размеров
- Найти dead code (неиспользуемые функции, импорты, файлы)
- Найти дубликаты логики
- Проверить все импорты — есть ли неиспользуемые
- Проверить типы — есть ли противоречия
- Проверить runtime баги (null access, необработанные promise rejection)

## Этап 2: Очистка dead code
- Удалить неиспользуемые экспорты
- Удалить неиспользуемые импорты
- Удалить закомментированный код
- Удалить дубликаты функций
- Проверить каждый файл: используется ли он кем-то?

## Этап 3: Оптимизация кода
- Объединить похожие функции
- Упростить сложные функции (>100 строк)
- Убрать дублирование между файлами
- Оптимизировать Prisma-запросы (не выбирать лишнее)
- Оптимизировать snapshot (не сериализовать лишнее)

## Этап 4: Структура проекта
- Сгруппировать файлы по назначению
- Вынести общие утилиты в helpers
- Упростить imports (barrel exports если нужно)
- Проверить naming conventions

## Этап 5: Финальная проверка
- lint + tsc + build
- Тест ключевых функций
- Commit + push

## Коммиты
- После каждого этапа — git commit
- После финала — push на GitHub
