# План: Сделать Мастера Игры настоящим профи D&D

## Проблемы текущего DM-контекста:
1. Нет памяти сюжета — DM видит только последние 15 сообщений
2. Способности не перечислены — DM не знает что игроки умеют
3. Таланты не видны — DM не знает выбранные таланты
4. Нет флагов сюжета — DM забывает ключевые события
5. Нет состояния подземелья — биом, этаж, зачищено ли
6. Предметы на земле не видны — DM не знает что можно подобрать
7. Короткая история — 15 сообщений, старые события теряются
8. Промпт можно улучшить — структура и инструкции

## Этапы (коммит после каждого):

### 1. Память сюжета (StoryMemory)
- Новая модель `StoryMemory` в Prisma: roomId, type (event/npc_met/choice/discovery), content, createdAt
- В dm-agent.ts: после разрешения действия, сохранять ключевые события
- В getDMContext: выводить последние 10 записей памяти сюжета

### 2. Способности + таланты в контексте
- В getDMContext: для каждого игрока вызывать computeAbilities()
- Выводить список способностей с источником (раса/класс/талант/свиток)
- Выводить выбранные таланты с описаниями

### 3. Флаги сюжета
- Новое поле `storyFlags String @default("")` на Room (JSON)
- DM может устанавливать флаги через success.storyFlags
- В getDMContext: выводить активные флаги

### 4. Состояние подземелья + предметы на земле
- В getDMContext: добавить биом, этаж, зачищено ли
- Выводить предметы на земле (playerName="__ground__")

### 5. Расширенная история чата
- Увеличить лимит с 15 до 30 сообщений
- Улучшить суммаризацию старых сообщений

### 6. Профессиональный промпт Мастера
- Полностью переписать SYSTEM_PROMPT_PLANNING
- Добавить правила: запоминать имена, отслеживать HP, не путать цели
- Добавить инструкции по ведению боя, диалогам, исследованию
- Добавить правила D&D 5e механик

### 7. Контекст действия игрока
- Перед действием игрока, добавить что он МОЖЕТ сделать
- Ближайшие цели, доступные предметы, видимые объекты

### 8. NPC отношения
- Отслеживать изменения disposition NPC
- Включать в контекст текущие отношения
