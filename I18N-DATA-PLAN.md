# План: Перевод игровых данных через функцию localize()

## Проблема:
Игровые данные (расы, классы, предметы, способности, монстры, заклинания) хранятся на русском. При смене языка UI переводится, но данные остаются на русском.

## Решение:
Создать функцию `localize(lang, key)` в i18n.ts, которая переводит игровые данные по ключу. Компоненты будут вызывать её вместо прямого отображения русского имени.

## Источники данных для перевода:
1. **presets.ts** — классы (12), расы (9), происхождения (10), стартовые предметы
2. **abilities.ts** — способности (~30)
3. **bestiary.ts** — монстры (51) — уже есть nameEn
4. **spellbook.ts** — заклинания (34) — уже есть nameEn
5. **item-database.ts** — предметы (108) — уже есть nameEn
6. **talent-data.ts** — таланты (120)
7. **conditions.ts** — состояния (10) — уже есть nameEn

## Стратегия:
- Создать словарь переводов в i18n.ts для каждого типа данных
- Добавить функцию `localizedName(lang, category, ruName)` 
- Компоненты используют её для отображения
- Для en: использовать nameEn где есть, переводить где нет
- Для es/de/fr/zh: переводить ключевые названия
