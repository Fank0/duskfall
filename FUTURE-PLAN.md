# DUSKFALL — План дальнейшего улучшения (v2.1 → v3.0)

*Создан на основе аудита текущего состояния (66 коммитов, 13 моделей, 19 компонентов, 20 API-маршрутов).*

---

## Текущее состояние (v2.0)

### ✅ Что есть и работает
- **Боевая система 2.0**: 10 условий, преимущество/помеха, спелл-слоты, AoE, фланги
- **Мир 2.0**: журнал квестов, карта мира (BSP), NPC+диалоги, день/ночь, погода, случайные встречи
- **Прогрессия**: дерево навыков (2 круга + ASI), экипировка (8 слотов), крафт (17 рецептов)
- **Визуал/UI**: анимации боя, токены с портретами, лог боя, эффекты сетки, 4 темы, 3 масштаба
- **Оптимизация**: кеш снапшотов, обрезка контекста LLM, кэш промптов, retry backoff, model routing
- **Инфраструктура**: React.memo, логирование, метрики, admin-API, валидация, sanitize, CI
- **Генератор подземелий**: 5 биомов, BSP, ловушки, боссы, глубина
- **LLM**: мультипровайдерная цепочка (GLM → Gemini → OpenRouter → Ollama → SDK)

### ❌ Что отсутствует (потеряно при сбросе или не реализовано)
- **Аккаунты и сохранения**: нет моделей Account/SaveSlot, нет auth-маршрутов, нет AuthScreen, нет MySavesDialog
- **i18n**: нет системы интернационализации (только русский)
- **Аудио**: нет процедурной музыки/SFX (Web Audio API)
- **Промежуточное звено**: нет socket.io в мини-сервисе (game-sync), только polling
- **Тестовое покрытие**: нет unit/integration/E2E тестов (CI проверяет только lint+tsc)
- **Безопасность сессий**: нет HMAC-подписанных cookies, нет bcrypt-хеширования паролей

## ФАЗА 6 — Аккаунты, Сохранения, Аудио, i18n (восстановление + доработка)

### 6.1 Система аккаунтов и сохранений 🔴 КРИТИЧЕСКИЙ
**Приоритет: КРИТИЧЕСКИЙ** | **Сложность: Высокая** | **Референс: BG3, Roll20**

Текущее состояние: игра работает без аккаунтов (гостевой режим). Нужно вернуть:
- Модель `Account` (id, username, passwordHash, createdAt)
- Модель `SaveSlot` (3 слота на аккаунт, привязка к Room + Player)
- API: `/api/auth/{register,login,logout,me}`, `/api/game/saves/{list,delete,update}`
- HMAC-подписанные httpOnly cookies (SESSION_SECRET env var)
- bcrypt 10 раундов для паролей
- Компоненты: `AuthScreen.tsx`, `MySavesDialog.tsx`
- Интеграция в `Lobby.tsx`: авторестор сессии при загрузке
- Валидация: логин 3-20 символов `[A-Za-z0-9_]`, пароль 8+
- Rate limiting: 3 рег/10мин, 5 логинов/10мин на IP
- Автопривязка слота при создании/входе в комнату
- Автопрокачка уровня при levelup

### 6.2 Процедурное аудио 🟡 ВЫСОКИЙ
**Приоритет: ВЫСОКИЙ** | **Сложность: Средняя** | **Референс: BG3, Disco Elysium**

Создать `src/lib/game/audio.ts` с Web Audio API:
- **Фоновая музыка** по настроению: peace (мир), combat (бой), tension (напряжение)
  - Каждое настроение = разные частоты, фильтры, темп
  - Плавный crossfade при смене настроения
  - Зависит от timeOfDay (ночь = tension) и combatActive (бой = combat)
- **SFX (звуковые эффекты)**:
  - `sfxDiceRoll()` — деревянный стук костей (BG3-style)
  - `sfxHit()` — удар (короткий низкий тон)
  - `sfxCrit()` — крит (восходящий аккорд)
  - `sfxMiss()` — промах (тихий шорох)
  - `sfxHeal()` — лечение (восходящий звон)
  - `sfxLevelUp()` — повышение уровня (торжественный аккорд)
  - `sfxConditionApply()` — наложение состояния (мягкий звон)
  - `sfxMonsterDeath()` — смерть монстра (низкий спад)
  - `sfxClick()` — клик UI (короткий щелчок)
  - `sfxError()` — ошибка (диссонанс)
- Все звуки = синусоидальные/треугольные волны с low-pass фильтром (НЕ резкие square/sawtooth)
- Настройки в `SettingsMenu.tsx`: громкость музыки, громкость SFX, вкл/выкл

### 6.3 Система интернационализации (i18n) 🟡 ВЫСОКИЙ
**Приоритет: ВЫСОКИЙ** | **Сложность: Средняя**

Создать `src/lib/game/i18n.ts` с поддержкой 6 языков:
- 🇷🇺 Русский (основной)
- 🇬🇧 English
- 🇪🇸 Español
- 🇩🇪 Deutsch
- 🇫🇷 Français
- 🇨🇳 中文

Структура:
- `t(lang, key)` — перевод строки
- `llmLangName(lang)` — имя языка для LLM-промпта
- `defaultLang()` — язык по умолчанию из Accept-Language
- Все UI-строки вынести в словари
- Выбор языка в `SettingsMenu.tsx` (флаги + названия)
- Язык сохраняется в zustand settings store + localStorage
- DM-промпт получает язык нарратива из settings

### 6.4 WebSocket (socket.io) для реального времени 🟡 ВЫСОКИЙ
**Приоритет: ВЫСОКИЙ** | **Сложность: Средняя**

Текущее состояние: только polling (5с/1.5с). Добавить real-time:
- Мини-сервис `mini-services/game-sync/` (port 3003) с socket.io
- События: `room:refresh` (состояние изменилось), `player:joined`, `player:left`, `combat:turn`
- Frontend: `src/lib/game/socket.ts` с `joinRoomSocket()`, `onRoomRefresh()`
- При получении `room:refresh` — обновлять snapshot без polling
- Polling остается как fallback (15с при стабильном сокете)
- Снижение задержки с 1.5-5с до <0.5с
