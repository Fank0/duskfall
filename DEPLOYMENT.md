# DUSKFALL — Руководство по развёртыванию (Вариант Б)

Это инструкция, как выложить игру на публичный адрес, чтобы друзья могли
подключиться из своих браузеров через интернет.

> **Кратко:** игра состоит из двух процессов — Next.js (порт 3000) и реле
> socket.io `game-sync` (порт 3003) для синхронизации в реальном времени.
> Шлюз Caddy спереди направляет запросы: с параметром `?XTransformPort=3003`
> — в реле, всё остальное — в Next.js. База данных — SQLite-файл на
> постоянном томе. Всё打包 в Docker Compose.

---

## Что нужно выбрать

| Платформа | Сложность | HTTPS | Цена | Подходит |
|-----------|-----------|-------|------|----------|
| **VPS + Docker Compose** (рекомендуется) | средняя | авто (Caddy) | ~$5/мес | ✅ полнофункционально |
| **Railway** | низкая | авто | ~$5/мес | ⚠️ два процесса нужно решить |
| **Render** | низкая | авто | ~$7/мес | ⚠️ нужен диск для SQLite |

Рекомендую **VPS + Docker Compose** — это самый прямой путь, всё работает
«из коробки» по той же схеме, что и песочница.

---

## Вариант 1. VPS + Docker Compose (рекомендуется)

### 1.1. Подготовьте сервер

1. Арендуйте VPS (Ubuntu 22.04+) у любого хостера: Hetzner, DigitalOcean,
   Vultr, Selectel, Timeweb и т.п. Подойдёт машина **2 ГБ RAM / 1 vCPU**.
2. Купите домен (необязательно, но нужно для HTTPS) и направьте его A-запись
   на IP вашего сервера. Подождите, пока DNS обновится (до часа).

### 1.2. Установите Docker на сервере

Зайдите на сервер по SSH и выполните:

```bash
# Установка Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# Выйдите и зайдите снова, чтобы группа применилась
exit
ssh user@ВАШ_IP
docker --version   # проверка
```

### 1.3. Загрузите код на сервер

Вариант A — через Git (если проект в репозитории):

```bash
git clone https://github.com/ВАШ_ЛОГИН/duskfall.git
cd duskfall
```

Вариант B — архивом с локальной машины:

```bash
# локально (у себя):
tar --exclude='node_modules' --exclude='.next' --exclude='db/*.db' \
    -czf duskfall.tar.gz -C /путь/к/проекту .
scp duskfall.tar.gz user@ВАШ_IP:~/
# на сервере:
mkdir -p ~/duskfall && cd ~/duskfall
tar -xzf ~/duskfall.tar.gz
```

### 1.4. Настройте переменные окружения

```bash
cd ~/duskfall
cp .env.example .env
nano .env
```

Заполните **обязательно**:

```
ZAI_API_KEY=ваш_ключ_от_z_ai_sdk
```

Без этого ключа ИИ-Мастер и генерация сцен работать не будут.
(При необходимости — `DUSKFALL_EMAIL` для уведомлений о сертификате HTTPS.)

### 1.5. Настройте домен (для HTTPS)

Откройте `Caddyfile.prod` и закомментируйте блок `:80`, раскомментируйте
блок с доменом, заменив `duskfall.example.com` на ваш домен:

```caddyfile
duskfall.example.com {
    @transform_port_query {
        query XTransformPort=3003
    }
    handle @transform_port_query {
        reverse_proxy app:3003
    }
    handle {
        reverse_proxy app:3000
    }
}
```

Caddy **автоматически** получит SSL-сертификат от Let's Encrypt при первом
запуске. Достаточно, чтобы домен уже указывал на IP сервера.

> Если домена нет — оставьте блок `:80`. Игра будет доступна по `http://ВАШ_IP`
> (без HTTPS; вебсокеты тоже работают по HTTP).

### 1.6. Запустите

```bash
docker compose up -d --build
```

Первый сборка займёт 3–7 минут. Затем:

```bash
docker compose ps          # оба сервиса Up
docker compose logs -f app # посмотреть логи
```

Откройте `http://ВАШ_ДОМЕН` (или `http://ВАШ_IP`) — должна открыться
лендинг-страница **DUSKFALL**.

### 1.7. Как играть с друзьями

1. Вы (хост): откройте сайт → **«Создать комнату»** → выберите народ/класс/
   происхождение → введите имя → **«Создать и войти»**.
2. Скопируйте **6-значный код комнаты** (кликните по коду в шапке справа —
   он скопируется в буфер).
3. Отправьте другу: ссылку на сайт + код комнаты.
4. Друг: открывает сайт → **«Войти по коду»** → вводит код, своё имя,選擇
   героя → **«Присоединиться»**.
5. Оба видят друг друга в «Отряде» и на сетке. Бой идёт по инициативе.

### 1.8. Обновление и обслуживание

```bash
# Обновить код после изменений:
cd ~/duskfall
git pull            # или заново загрузить архив
docker compose up -d --build

# Остановить:
docker compose down

# Резервная копия базы (состояние всех комнат):
docker compose cp app:/data/custom.db ./backup-$(date +%F).db

# Сбросить все комнаты к пустоте (осторожно!):
docker compose down
docker volume rm duskfall_duskfall-data
docker compose up -d --build
```

## Вариант 2. Railway

Railway умеет два процесса через `Procfile`. Создайте в корне `Procfile`:

```
web: ./start.sh
```

Затем:

1. Создайте проект на [railway.app](https://railway.app), подключите репозиторий.
2. Variables: добавьте `ZAI_API_KEY`, `DATABASE_URL=file:/data/custom.db`,
   `PORT=3000`, `SYNC_PORT=3003`.
3. Settings → Networking → сгенерируйте домен (например `duskfall.up.railway.app`).
4. Добавьте **Volume** (переменная `volumeMountPath=/data`) — это persistent
   storage для SQLite.
5. ⚠️ Проблема: Railway пробрасывает только ОДИН порт наружу (`PORT`), а
   реле socket.io слушает 3003. Решение: настроить в Railway только порт
   `3000` для web, а socket.io повесить на тот же порт — для этого
   используйте `Caddyfile.prod`-подобный роутинг через Railway **не
   получится** (Railway не делает path-based роутинг query-параметров).

   **Простое решение для Railway:** поднимите реле socket.io на том же
   порту 3000 внутри процесса. Откройте `mini-services/game-sync/index.ts`
   и поменяйте `const PORT = 3003;` на `const PORT = 3000;` — НО тогда
   возникнет конфликт портов с Next.js. Поэтому на Railway лучше
   интегрировать socket.io прямо в Next.js (см. раздел «Альтернатива»
   ниже) или использовать схему из Варианта 1.

Из-за сложности с двумя портами **для мультиплеера рекомендуем VPS**.

## Вариант 3. Render

1. Создайте **Web Service** на [render.com](https://render.com), подключите репозиторий.
2. Runtime: **Docker**. Build Command: _(оставьте пусто, использует Dockerfile)_.
3. Environment variables: `ZAI_API_KEY`, `DATABASE_URL=file:/data/custom.db`.
4. Добавьте **Disk** (mount path `/data`, размер 1 ГБ) для SQLite.
5. ⚠️ Та же проблема с двумя портами, что и в Railway. Render пробрасывает
   один порт. Применимы те же рекомендации.

## Альтернатива: один порт (упрощает PaaS)

Если хостинг пробрасывает только один порт, можно интегрировать socket.io
**внутри** Next.js-процесса, чтобы и web, и вебсокеты жили на `:3000`.
Для этого нужно поднять кастомный сервер. Это требует доработки проекта
(обратитесь к разработчику). Текущая двухпроцессная схема оптимальна для
VPS, где порты не ограничены.

## Переменные окружения — справка

| Переменная | Назначение | Обязательно |
|------------|------------|-------------|
| `ZAI_API_KEY` | Ключ z-ai SDK (LLM-Мастер, генерация сцен) | ✅ да |
| `DATABASE_URL` | Путь к БД. SQLite: `file:/data/custom.db` | да (по умолчанию в Dockerfile) |
| `PORT` | Порт Next.js | 3000 (по умолчанию) |
| `SYNC_PORT` | Порт реле socket.io | 3003 (по умолчанию) |
| `DUSKFALL_EMAIL` | Email для уведомлений Let's Encrypt | только для домена |

## Переход на PostgreSQL (опционально)

SQLite на постоянном томе достаточно для группы из 2–8 друзей. Если хотите
масштабироваться или использовать managed-БД:

1. В `prisma/schema.prisma` поменяйте провайдер:
   ```prisma
   datasource db {
     provider = "postgresql"
     url      = env("DATABASE_URL")
   }
   ```
2. В `.env` укажите `DATABASE_URL=postgresql://user:pass@host:5432/duskfall`.
3. Перегенерируйте клиент и примените схему:
   ```bash
   bunx prisma generate
   bunx prisma db push
   ```
4. Пересоберите образ.

## Проверка после деплоя

Откройте сайт и убедитесь:

- [ ] Лендинг DUSKFALL загружается, нет ошибок в консоли браузера (F12).
- [ ] Создание комнаты работает — выдаётся 6-значный код.
- [ ] Сцена-картинка леса отображается (значит `ZAI_API_KEY` валиден).
- [ ] Откройте комнату во второй вкладке/устройстве, войдите по коду —
      второй герой появляется в «Отряде» в реальном времени (значит
      socket.io-реле и шлюз работают).
- [ ] Атакуйте врага — начинается бой, бросается инициатива, ходы чередуются.

Если вторая вкладка не видит обновлений в реальном времени — проблема в
роутинге вебсокетов через шлюз. Проверьте, что Caddy направляет
`?XTransformPort=3003` на порт 3003 контейнера `app` (см. `Caddyfile.prod`).

## Частые проблемы

**«Мастер не отвечает» / сцены не генерируются.** Не задан или неверен
`ZAI_API_KEY`. Проверьте `docker compose exec app printenv ZAI_API_KEY`.

**Вторая вкладка не получает обновления.** Шлюз не маршрутизирует
вебсокеты. Убедитесь, что подключение идёт через `/?XTransformPort=3003`
(это делает фронтенд автоматически) и Caddy пробрасывает его на порт 3003.

**«Комната не найдена» после перезапуска.** База не на постоянном томе —
данные стираются. Проверьте, что `DATABASE_URL=file:/data/custom.db` и том
`duskfall-data` примонтирован (`docker volume ls`).

**Сертификат HTTPS не выдаётся.** Домен ещё не указывает на IP сервера, либо
порт 443 закрыт файрволом. Откройте 80/443: `sudo ufw allow 80,443/tcp`.
